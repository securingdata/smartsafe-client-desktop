First initialization

Lors de la première utilisation du token ou après une mise à jour du token, une première personnalisation doit être faite. Cette action se fait en sélectionnant l'entrée du menu 'Fichier' → 'Initialisation'.

Le seul champ obligatoire est le 'Mot de passe utilisateur'. Ce mot de passe est celui qui protège les données enregistrées dans le token. Il est à noter que ce mot de passe peut être changé à tout moment (il faut être au préalable authentifié) avec l'entrée du menu 'Fichier' → 'Changer le mot de passe'.

Si vous voulez également initialiser le token avec une sauvegarde précédente de vos données, il faut activer la boîte 'Utiliser cette sauvegarde', donner le chemin vers le fichier de sauvegarde ainsi que son mot de passe associé. Cette action ne peut être faite que sur un token non personnalisé. Un token est considéré comme personnalisé d√®s qu'un mot de passe est enregistré.

Enfin, lancez le processus de personnalisation en cliquant sur le boutton 'Personnaliser'. Quand le message 'Fait !' s'affiche, le token est prêt à être utilisé.
